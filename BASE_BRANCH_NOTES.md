# Base 分支说明

## 状态描述

本分支为优化前的基准版本（base branch），记录了当前代码库的主要状态：

### 1. 板块列表数据
- **状态**：基本稳定
- **获取方式**：通过 `scripts/fetch_sector.py` 使用 20 维度并发扫描东方财富板块
- **覆盖范围**：行业板块、概念板块、地域板块
- **唯一性问题**：通过多排序字段（f12, f3, f2, f6, f5, f4, f17, f18, f8, f10, f15, f16, f11, f9, f23, f20, f21, f22, f24, f25）的正反向扫描，确保覆盖所有板块

### 2. 全量历史数据下载
- **状态**：速度极慢，存在严重性能问题
- **原因分析**：
  1. 每只股票需要调用 2 次 baostock API（K 线数据 + 复权因子）
  2. 串行处理，即使有 20 个并发任务，每个任务内部仍是串行
  3. baostock 免费 API 有隐式速率限制
  4. 获取全量历史（2005-2026）数据量大，单次请求耗时 30-50 秒

- **当前性能**：
  - 单只股票下载时间：约 50 秒
  - 5000 只股票总耗时：约 3.5 小时（20 并发）
  - 复权因子获取时间：占总时间的 40-50%

### 3. 待优化方向
- 批量获取复权因子（减少 50% API 调用）
- 切换到更快的数据源（如 akshare、Tushare）
- 增加任务内部并发度
- 自建缓存层
